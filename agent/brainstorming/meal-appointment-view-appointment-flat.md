# 식사 약속 조회 사용자 명세 평탄화
1. 공유 약속 링크(`/appointments/{appointmentId}`)를 아는 누구나 로그인 없이 약속 정보를 조회할 수 있어야 한다.
2. 이번 조회 명세에는 응답 수정 기능이 포함되지 않으며, 닉네임·선택적 PIN은 추후 응답 제출 플로우에서 사용된다는 점을 명확히 안내한다.
3. 약속이 존재하지 않으면 약속을 찾을 수 없다는 안내만 제공하고 추가 조치를 요구하지 않는다.
4. 조회 시 약속 제목과 요약 설명을 확인하여 약속의 주제와 목적을 이해할 수 있어야 한다.
5. 사용자는 선택된 시간 슬롯 템플릿의 이름과 설명을 확인해 슬롯 구성 규칙을 파악할 수 있어야 한다.
6. 약속 생성 시각 등 메타데이터는 참고용으로 제공하지만 응답 제출 흐름과 직접 연동되지는 않는다.
7. 시스템은 저장된 템플릿 규칙을 적용해 주간 기본 템플릿의 슬롯 목록을 동적으로 산출해야 한다.
8. 슬롯 목록은 날짜와 식사 시간대 기준으로 정렬되어 사용자가 가능한 시간을 쉽게 파악할 수 있어야 한다.
9. 템플릿에서 파생된 슬롯만 표시되고 사용자가 수동으로 추가한 슬롯은 존재하지 않는다는 점을 분명히 해야 한다.
10. 닉네임을 등록하지 않은 사용자는 약속 정보만 확인할 수 있으며 이번 명세에서는 추가 입력이나 응답 제출을 요구하지 않는다.
11. 조회 화면은 기존 응답자의 가용성 선택을 수정하거나 재제출하도록 유도하지 않으며 읽기 전용으로 유지된다.
12. 닉네임 중복 또는 PIN 불일치 등 검증 흐름은 응답 수정 기능과 함께 별도 범위임을 명확히 한다.
13. 각 슬롯별 가능한 인원 수와 응답률 요약은 모든 사용자가 동일하게 확인할 수 있어야 한다.
14. 응답자 리스트와 각 응답자의 세부 선택 정보를 누구나 열람할 수 있도록 공개해야 한다.
15. 요약 데이터는 최신 응답을 반영해야 하며 사용자가 필요 시 데이터를 새로 고칠 수 있음을 알려야 한다.
16. 서버나 네트워크 오류가 발생하면 일시적 문제임을 전달하고 나중에 다시 시도하도록 안내해야 한다.
17. 템플릿 로딩 실패와 같은 예외 상황에서도 사용자 데이터 손실이 없음을 알리고 재시도로 정상 슬롯 목록을 확인할 수 있음을 안내해야 한다.
