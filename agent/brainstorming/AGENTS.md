# 브레인스토밍 작업 지침

이 디렉터리의 지침은 브레인스토밍 단계에서 생성하는 모든 파일(`agent/brainstorming/*`)에 적용됩니다.

1. 작업 흐름은 **명세 → 평탄화 → 체크리스트** 순서를 반드시 따릅니다.
   - 사용자의 요구사항을 이해한 뒤, 요구사항의 언어로 명세 파일을 작성합니다.
   - 작성한 명세를 기반으로 동일한 내용을 누락 없이 평탄화한 파일을 생성합니다.
   - 평탄화된 내용을 체크리스트로 변환하여 새로운 파일에 저장하며, 각 항목에는 고유 번호를 부여합니다.
2. 체크리스트 항목은 생성 이후 어떠한 경우에도 수정하거나 삭제하지 않습니다. 추가 사항은 항목 번호를 이어서 새 항목으로만 작성합니다.
3. 사용자의 지시에 따라 각 체크리스트 항목에 승인/반려 여부를 명시하며, 반려 사유가 있으면 함께 기록합니다.
4. 체크리스트에 새로운 항목을 추가할 때에도 기존 형식을 유지하고 번호는 연속되게 증가시킵니다.
5. 파일 작성 시 필요하다면 작업 히스토리를 명확히 하기 위해 타임스탬프나 버전 표기를 사용할 수 있으나, 본문 내용은 변경하지 않습니다.
