# 식사 약속 조회 사용자 체크리스트
1. (상태: 미확인) 공유 링크(`/appointments/{appointmentId}`)로 누구나 로그인 없이 약속 정보를 조회할 수 있는지 확인한다.
2. (상태: 미확인) 기존 닉네임(+선택적 PIN) 사용자가 응답 수정을 위해 동일 정보를 다시 입력하도록 요구하는지 확인한다.
3. (상태: 미확인) 약속이 존재하지 않을 때 "약속을 찾을 수 없음" 안내만 제공하고 추가 행동을 요구하지 않는지 확인한다.
4. (상태: 미확인) 조회 화면에서 약속 제목과 요약 설명을 제공해 약속 목적을 이해할 수 있는지 확인한다.
5. (상태: 미확인) 선택된 시간 슬롯 템플릿의 이름과 설명이 표시되어 슬롯 구성 규칙을 파악할 수 있는지 확인한다.
6. (상태: 미확인) 약속 생성 시각 등의 메타데이터가 참고용으로 제공되지만 응답 흐름과 직접 연동되지 않는지 확인한다.
7. (상태: 미확인) 시스템이 템플릿 규칙을 적용해 주간 기본 템플릿 슬롯 목록을 동적으로 생성하는지 확인한다.
8. (상태: 미확인) 슬롯 목록이 날짜와 식사 시간대 기준으로 정렬되어 있는지 확인한다.
9. (상태: 미확인) 템플릿에서 파생된 슬롯만 표시되고 수동 추가 슬롯이 없음을 명확히 알리는지 확인한다.
10. (상태: 미확인) 닉네임을 등록하지 않은 사용자가 약속 정보만 확인하고 응답 제출 전 닉네임(및 PIN) 입력이 필요함을 안내하는지 확인한다.
11. (상태: 미확인) 기존 응답자가 동일 닉네임(+PIN) 입력 시 이전 가용성 선택이 로드되는지 확인한다.
12. (상태: 미확인) 닉네임 중복 또는 PIN 불일치 시 재시도 또는 새 닉네임 등록 안내를 제공하는지 확인한다.
13. (상태: 미확인) 각 슬롯별 가능한 인원 수와 응답률 요약이 모든 사용자에게 동일하게 제공되는지 확인한다.
14. (상태: 미확인) 응답자 개인 정보는 숨기고 집계 데이터만 제공하여 개인정보 보호를 준수하는지 확인한다.
15. (상태: 미확인) 요약 데이터가 최신 응답을 반영하고 필요 시 데이터 새로고침 방법을 안내하는지 확인한다.
16. (상태: 미확인) 서버 또는 네트워크 오류 시 일시적 문제 안내와 재시도 권장을 제공하는지 확인한다.
17. (상태: 미확인) 템플릿 로딩 실패 등 예외 상황에서 사용자 데이터가 안전하며 재시도로 슬롯 목록을 확인할 수 있음을 알리는지 확인한다.
