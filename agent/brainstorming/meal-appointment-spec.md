# 식사 약속 조율 앱 명세

## 1. 개요
- 목적: 사용자들이 식사 약속을 간편하게 생성하고, 공유하며, 각자의 가능 시간을 토대로 최적의 시간 슬롯을 확정하도록 돕는다.
- 플랫폼 가정: 모바일 웹과 데스크톱 웹 모두에서 사용 가능한 반응형 웹 애플리케이션.
- 사용자 식별: 개인 식별 정보를 수집하지 않으며, 사용자는 임시 닉네임 또는 기기 기반 세션으로 구분한다.

## 2. 주요 기능
### 2.1 약속 생성 및 공유
- 생성자는 약속 제목, 설명, 식사 유형(조식/중식/석식 등), 유효 기간, 기본 규칙 세트를 입력한다.
- 약속 생성 시 자동으로 참여 링크와 초대 코드가 발급된다.
- 생성자는 링크 복사, QR, 메시지 앱 공유 등 간편 공유 수단을 선택할 수 있다.

### 2.2 규칙 기반 시간 슬롯 산출
- 규칙 유형 예시: 요일 반복, 특정 날짜 범위, 식사 시간대(예: 11:30~13:30), 시간 간격(예: 30분 단위).
- 앱은 규칙에 따라 시간 슬롯 후보를 자동 생성하고, 필요한 경우 생성자가 수동으로 슬롯을 추가/삭제할 수 있다.
- 생성된 슬롯은 타임존 정보를 포함하며 모든 참여자에게 동일하게 노출된다.

### 2.3 약속 참여 및 참여자 관리
- 초대 링크 또는 코드로 접근한 사용자는 닉네임을 입력하고 약속에 참여한다.
- 참여자는 자신의 가능 여부(가능/불가능/미정)와 선호도를 각 슬롯별로 선택할 수 있다.
- 참여자는 언제든지 자신의 응답을 수정하거나 갱신할 수 있다.
- 생성자는 참여자 목록을 확인하고 필요 시 특정 참여자의 응답을 잠금 또는 삭제할 수 있다.

### 2.4 가용 현황 시각화
- 각 슬롯별 총 참여 가능 인원, 미정 인원 수, 응답률을 시각화한다.
- 참여자 개인은 자신의 응답과 전체 현황을 동시에 확인할 수 있다.
- 생성자는 가중치를 기준으로 가장 유력한 시간 슬롯을 추천받거나, 수동으로 확정 상태로 변경할 수 있다.
- 약속이 확정되면 모든 참여자에게 확정된 시간과 장소를 표시한다.

### 2.5 알림 및 업데이트
- 변경 사항(새로운 참여자, 응답 수정, 확정 등)은 앱 내 알림과 선택적 브라우저 푸시로 전달한다.
- 알림 설정은 약속 단위로 on/off 토글만 제공하여 단순성을 유지한다.

## 3. 데이터 및 개인정보 보호
- 사용자 입력 정보: 닉네임, 응답 상태, 선택적 코멘트 등 개인을 특정할 수 없는 항목만 허용한다.
- 서버는 세션 토큰과 약속/슬롯/응답 데이터만 저장하며, 이메일/전화번호 등 식별 정보는 요구하지 않는다.
- 데이터 보존 정책: 약속 종료 후 30일이 지나면 자동 삭제되도록 설정하여 개인정보 노출을 최소화한다.

## 4. 사용성 우선 설계
- 로그인 없이 참여 가능하며, 생성자도 최소 입력만으로 약속을 만들 수 있다.
- 인터페이스는 모바일 기준으로 최적화하고, 핵심 흐름(생성 → 공유 → 응답 → 확정)은 3단계 이내 클릭으로 구성한다.
- 복잡한 보안 절차(2FA 등)는 제공하지 않되, 기본적인 링크 난수화와 세션 만료는 유지한다.

## 5. 기술 및 운영 고려
- 프런트엔드: React 또는 유사한 SPA 프레임워크, 반응형 UI 라이브러리 활용.
- 백엔드: 서버리스 함수 또는 경량 Node.js 서비스로 약속/슬롯/응답 API 제공.
- 실시간 현황 갱신을 위해 웹소켓 또는 짧은 주기의 폴링을 사용.
- 배포는 HTTPS 지원이 가능한 관리형 플랫폼을 활용하며, 링크 공유를 위한 맞춤 도메인 사용을 고려한다.

## 6. 향후 확장 아이디어

- 선호도에 따른 자동 추천 알고리즘 고도화.
- 그룹 채팅 또는 코멘트 스레드 기능 도입.
