# 식사 약속 조율 앱 명세

## 1. 개요
- 목적: 사용자들이 식사 약속을 간편하게 생성하고 공유하며, 각자의 가능 시간을 바탕으로 조율한다.
- 플랫폼 가정: 모바일 웹과 데스크톱 웹 모두에서 사용 가능한 반응형 웹 애플리케이션.
- 사용자 식별: 개인 식별 정보를 수집하지 않으며, 사용자는 약속마다 입력한 닉네임으로만 구분된다.

## 2. 주요 기능
### 2.1 약속 생성 및 공유
- 생성자는 약속 제목, 간단한 설명, 시간 슬롯 생성을 위한 규칙 세트를 입력한다.
- 약속 생성 시 공유용 링크가 자동 발급되며, 초대 코드는 사용하지 않는다.
- 생성자는 발급된 링크를 복사해 원하는 채널에 직접 붙여넣는 방식으로 약속을 배포한다.

### 2.2 규칙 기반 시간 슬롯 산출
- 시간 슬롯은 입력된 규칙을 바탕으로 자동 생성되며, 구체적인 규칙 유형과 옵션은 이해관계자와 협의해 추후에 결정한다.
- 자동 생성된 시간 슬롯은 시스템이 일관되게 관리하며, 사용자에 의한 수동 추가나 삭제는 지원하지 않는다.

### 2.3 약속 참여 및 가용 응답
- 공유 링크로 접근한 사용자는 닉네임을 입력해 약속에 참여하며 필요 시 본인 확인을 위해 짧은 비밀번호를 선택적으로 설정할 수 있다.
- 참여자는 참여 가능한 시간 슬롯을 선택하여 표시하고, 선택하지 않은 슬롯은 자동으로 불가능 처리된다.
- 참여자는 자신의 선택 내역을 언제든지 수정하거나 갱신할 수 있다.

### 2.4 가용 현황 시각화
- 각 시간 슬롯별로 선택한 참여자 수, 응답률 등의 지표를 시각화하여 제공한다.
- 참여자는 자신의 선택과 전체 현황을 동시에 확인할 수 있다.
- 앱은 확정이나 잠금과 같은 추가 상태를 제공하지 않으며, 공유된 가용성 정보를 기반으로 참여자들이 직접 조율하도록 한다.

## 3. 데이터 및 개인정보 보호
- 저장 데이터는 약속 정보, 시간 슬롯 정보, 참여자가 선택한 슬롯, 닉네임, 선택적으로 설정된 짧은 비밀번호(안전한 방식으로 저장)에 한정된다.
- 세션 토큰과 같은 추가 식별 정보는 저장하지 않는다.
- 이메일, 전화번호 등 개인을 식별할 수 있는 정보는 수집하지 않는다.
- 데이터 자동 삭제 기능은 제공하지 않으며, 보존 정책은 추후 논의한다.

## 4. 사용성 우선 설계
- 로그인 없이 약속을 생성하고 참여할 수 있도록 최소 입력 흐름을 유지한다.
- 인터페이스는 모바일 사용성을 우선 고려해 단순하고 직관적인 구성으로 설계한다.
- 고급 보안 절차보다 접근성과 사용 편의성을 우선하지만, 공유 링크 자체는 난수화하여 추측을 어렵게 한다.
