# 식사 약속 조율 앱 체크리스트
1. (상태: 승인 대기) 앱이 식사 약속 생성, 공유, 시간 조율을 위한 반응형 웹 애플리케이션인지 확인한다.
2. (상태: 반려 - 사용자는 닉네임으로만 식별) 사용자가 개인정보 대신 임시 닉네임이나 기기 세션으로 식별되는지 확인한다.
3. (상태: 반려 - 식사 유형과 유효 기간 제외) 생성자가 약속 제목, 설명, 식사 유형, 유효 기간, 규칙 세트를 입력하여 약속을 생성할 수 있는지 확인한다.
4. (상태: 반려 - 초대 코드 제공하지 않음) 약속 생성 시 참여 링크와 초대 코드가 자동 발급되는지 확인한다.
5. (상태: 반려 - 링크 복사 방식만 허용) 생성자가 링크 복사, QR, 메시지 공유 방식으로 약속을 배포할 수 있는지 확인한다.
6. (상태: 반려 - 규칙 세부 옵션은 추후 결정) 시간 슬롯이 요일 반복, 날짜 범위, 식사 시간대, 시간 간격 규칙으로 자동 생성되는지 확인한다.
7. (상태: 반려 - 수동 추가/삭제 불가) 생성자가 자동 생성된 시간 슬롯을 수동으로 추가 또는 삭제할 수 있는지 확인한다.
8. (상태: 반려 - 불필요한 맥락) 모든 시간 슬롯이 타임존 정보를 포함해 참여자에게 동일하게 제공되는지 확인한다.
9. (상태: 반려 - 초대 코드를 사용하지 않음) 초대 링크나 코드로 들어온 사용자가 닉네임을 입력하고 약속에 참여할 수 있는지 확인한다.
10. (상태: 반려 - 가능/불가능 2상태만 사용) 참여자가 각 시간 슬롯에 대해 가능, 불가능, 미정, 선호도를 표시할 수 있는지 확인한다.
11. (상태: 승인 대기) 참여자가 자신의 응답을 언제든지 수정하거나 갱신할 수 있는지 확인한다.
12. (상태: 반려 - 불필요한 기능) 생성자가 참여자 목록을 확인하고 응답을 잠금 또는 삭제할 수 있는지 확인한다.
13. (상태: 승인 대기) 앱이 슬롯별 가능 인원, 미정 인원, 응답률을 시각화하는지 확인한다.
14. (상태: 승인 대기) 참여자가 개인 응답과 전체 현황을 동시에 확인할 수 있는지 확인한다.
15. (상태: 반려 - 확정 기능 제공하지 않음) 생성자가 가중치 기반 추천이나 수동 선택으로 확정 시간 슬롯을 결정할 수 있는지 확인한다.
16. (상태: 반려 - 확정 관련 기능 없음) 확정된 시간과 장소가 모든 참여자에게 표시되는지 확인한다.
17. (상태: 반려 - 알림 기능 제외) 앱이 변경 사항을 알림과 선택적 푸시로 전달하는지 확인한다.
18. (상태: 반려 - 알림 토글 제공하지 않음) 알림 설정이 약속 단위 on/off 토글로 제공되는지 확인한다.
19. (상태: 반려 - 세션 토큰 제외, 선택적 비밀번호 허용) 저장 데이터가 비식별 정보와 세션 토큰, 약속/슬롯/응답 데이터에 한정되는지 확인한다.
20. (상태: 승인 대기) 이메일이나 전화번호 등 개인 식별 정보를 수집하지 않는지 확인한다.
21. (상태: 반려 - 데이터 자동 삭제 제공하지 않음) 약속 종료 30일 후 데이터가 자동 삭제되는지 확인한다.
22. (상태: 승인 대기) 로그인 없이 약속 생성 및 참여가 가능한지 확인한다.
23. (상태: 승인 대기) 핵심 흐름이 3단계 이내 클릭으로 처리되도록 설계되었는지 확인한다.
24. (상태: 반려 - 불필요한 맥락) 보안이 기본 수준(링크 난수화, 세션 만료)으로 유지되면서 사용성을 우선하는지 확인한다.
25. (상태: 반려 - 불필요한 맥락) 프런트엔드가 React 기반 SPA와 반응형 UI 라이브러리를 사용하는지 확인한다.
26. (상태: 반려 - 불필요한 맥락) 백엔드가 서버리스 또는 경량 Node.js 서비스로 약속/슬롯/응답 API를 제공하는지 확인한다.
27. (상태: 반려 - 불필요한 맥락) 실시간 현황 갱신을 위해 웹소켓 또는 짧은 주기 폴링을 사용하는지 확인한다.
28. (상태: 반려 - 불필요한 맥락) 배포가 HTTPS 지원 관리형 플랫폼과 맞춤 도메인을 이용하는지 확인한다.
29. (상태: 반려 - 불필요한 기능) 향후 확장 기능에 캘린더 내보내기, 추천 알고리즘 고도화, 그룹 채팅을 고려하는지 확인한다.
30. (상태: 승인 대기) 사용자가 약속마다 입력한 닉네임으로만 식별되는지 확인한다.
31. (상태: 승인 대기) 생성자가 약속 제목, 간단한 설명, 시간 슬롯 규칙 세트만으로 약속을 생성하는지 확인한다.
32. (상태: 승인 대기) 약속 생성 시 공유용 링크가 자동 발급되고 초대 코드는 생성되지 않는지 확인한다.
33. (상태: 승인 대기) 생성자가 공유 링크를 복사해 원하는 채널에 직접 붙여넣는 방식으로 약속을 배포하는지 확인한다.
34. (상태: 승인 대기) 시간 슬롯 규칙의 구체적인 유형과 옵션이 추후 협의 사항으로 남겨져 있는지 확인한다.
35. (상태: 승인 대기) 시간 슬롯이 규칙에 따라 자동 생성되고 수동 추가나 삭제가 불가능한지 확인한다.
36. (상태: 승인 대기) 사용자가 공유 링크로 접속해 닉네임을 입력하는 방식으로 약속에 참여하는지 확인한다.
37. (상태: 승인 대기) 참여자가 가능한 시간 슬롯을 선택해 표시하고 선택하지 않은 슬롯은 불가능으로 처리되는지 확인한다.
38. (상태: 승인 대기) 참여자가 자신의 선택 내역을 언제든지 수정할 수 있는지 확인한다.
39. (상태: 승인 대기) 앱이 확정이나 잠금과 같은 추가 상태를 제공하지 않고 가용성 공유만 지원하는지 확인한다.
40. (상태: 반려 - 시간 슬롯 저장하지 않음, 템플릿만 저장) 저장 데이터가 약속 정보, 시간 슬롯, 선택된 슬롯, 닉네임, 선택적 짧은 비밀번호로 한정되는지 확인한다.
41. (상태: 승인 대기) 시스템이 세션 토큰을 저장하지 않고 데이터 자동 삭제 기능을 제공하지 않는지 확인한다.
42. (상태: 반려 - 다시 제안하지 말 것) 고급 보안 절차보다 접근성과 사용 편의성을 우선하면서 공유 링크를 난수화하는지 확인한다.
43. (상태: 승인 대기) 시스템이 약속별로 선택된 시간 슬롯 템플릿만 저장하고 실제 슬롯은 규칙 기반으로 동적으로 생성하는지 확인한다.
