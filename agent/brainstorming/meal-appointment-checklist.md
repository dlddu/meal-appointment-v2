# 식사 약속 조율 앱 체크리스트
1. (상태: 승인 대기) 앱이 식사 약속 생성, 공유, 시간 조율을 위한 반응형 웹 애플리케이션인지 확인한다.
2. (상태: 승인 대기) 사용자가 개인정보 대신 임시 닉네임이나 기기 세션으로 식별되는지 확인한다.
3. (상태: 승인 대기) 생성자가 약속 제목, 설명, 식사 유형, 유효 기간, 규칙 세트를 입력하여 약속을 생성할 수 있는지 확인한다.
4. (상태: 승인 대기) 약속 생성 시 참여 링크와 초대 코드가 자동 발급되는지 확인한다.
5. (상태: 승인 대기) 생성자가 링크 복사, QR, 메시지 공유 방식으로 약속을 배포할 수 있는지 확인한다.
6. (상태: 승인 대기) 시간 슬롯이 요일 반복, 날짜 범위, 식사 시간대, 시간 간격 규칙으로 자동 생성되는지 확인한다.
7. (상태: 승인 대기) 생성자가 자동 생성된 시간 슬롯을 수동으로 추가 또는 삭제할 수 있는지 확인한다.
8. (상태: 승인 대기) 모든 시간 슬롯이 타임존 정보를 포함해 참여자에게 동일하게 제공되는지 확인한다.
9. (상태: 승인 대기) 초대 링크나 코드로 들어온 사용자가 닉네임을 입력하고 약속에 참여할 수 있는지 확인한다.
10. (상태: 승인 대기) 참여자가 각 시간 슬롯에 대해 가능, 불가능, 미정, 선호도를 표시할 수 있는지 확인한다.
11. (상태: 승인 대기) 참여자가 자신의 응답을 언제든지 수정하거나 갱신할 수 있는지 확인한다.
12. (상태: 승인 대기) 생성자가 참여자 목록을 확인하고 응답을 잠금 또는 삭제할 수 있는지 확인한다.
13. (상태: 승인 대기) 앱이 슬롯별 가능 인원, 미정 인원, 응답률을 시각화하는지 확인한다.
14. (상태: 승인 대기) 참여자가 개인 응답과 전체 현황을 동시에 확인할 수 있는지 확인한다.
15. (상태: 승인 대기) 생성자가 추천 또는 수동 선택을 통해 확정 시간 슬롯을 결정할 수 있는지 확인한다.
16. (상태: 승인 대기) 확정된 시간과 장소가 모든 참여자에게 표시되는지 확인한다.
17. (상태: 승인 대기) 앱이 변경 사항을 알림과 선택적 푸시로 전달하는지 확인한다.
18. (상태: 승인 대기) 알림 설정이 약속 단위 on/off 토글로 제공되는지 확인한다.
19. (상태: 승인 대기) 저장 데이터가 비식별 정보와 세션 토큰, 약속/슬롯/응답 데이터에 한정되는지 확인한다.
20. (상태: 승인 대기) 이메일이나 전화번호 등 개인 식별 정보를 수집하지 않는지 확인한다.
21. (상태: 승인 대기) 약속 종료 30일 후 데이터가 자동 삭제되는지 확인한다.
22. (상태: 승인 대기) 로그인 없이 약속 생성 및 참여가 가능한지 확인한다.
23. (상태: 승인 대기) 핵심 흐름이 3단계 이내 클릭으로 처리되도록 설계되었는지 확인한다.
24. (상태: 승인 대기) 보안이 기본 수준(링크 난수화, 세션 만료)으로 유지되면서 사용성을 우선하는지 확인한다.
25. (상태: 승인 대기) 프런트엔드가 React 기반 SPA와 반응형 UI 라이브러리를 사용하는지 확인한다.
26. (상태: 승인 대기) 백엔드가 서버리스 또는 경량 Node.js 서비스로 약속/슬롯/응답 API를 제공하는지 확인한다.
27. (상태: 승인 대기) 실시간 현황 갱신을 위해 웹소켓 또는 짧은 주기 폴링을 사용하는지 확인한다.
28. (상태: 승인 대기) 배포가 HTTPS 지원 관리형 플랫폼과 맞춤 도메인을 이용하는지 확인한다.
29. (상태: 승인 대기) 향후 확장 기능에 캘린더 내보내기, 추천 알고리즘 고도화, 그룹 채팅을 고려하는지 확인한다.
