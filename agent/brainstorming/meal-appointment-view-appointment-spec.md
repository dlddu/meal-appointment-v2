# 식사 약속 조회 사용자 브레인스토밍 명세

## 1. 개요
- 목적: 공유된 약속 링크를 통해 생성자와 초대받은 사용자가 약속 정보를 확인하고, 자신의 응답 현황 및 전체 가용 요약을 파악할 수 있도록 최종 사용자 관점의 요구를 정의한다.
- 범위: 로그인 없이 접근 가능한 약속 상세 조회 흐름 전반을 포함하며, 화면 배치나 스타일과 같은 UI/UX 세부사항은 제외한다.

## 2. 조회 진입 조건
- 약속 조회는 `/appointments/{appointmentId}` 형태의 공유 링크를 통해 수행되며 링크를 알고 있는 누구나 접근할 수 있다.
- 최초 접근 시 인증 절차는 없으며, 닉네임·선택적 PIN을 이미 등록한 사용자가 응답을 수정하려면 동일한 정보를 다시 입력해야 한다.
- 약속이 존재하지 않거나 삭제된 경우 사용자에게 약속을 찾을 수 없음을 알리고 추가 조치를 요구하지 않는다.

## 3. 약속 기본 정보 확인
- 약속 제목과 요약 설명이 조회되어 약속의 주제와 목적을 이해할 수 있어야 한다.
- 약속 생성 시 선택된 시간 슬롯 템플릿의 이름과 설명을 확인하여 어떤 규칙으로 슬롯이 구성되는지 파악할 수 있어야 한다.
- 약속 생성 시각 등 메타데이터는 참고용으로 제공하되, 응답 제출과 직접적인 연동은 요구하지 않는다.

## 4. 시간 슬롯 정보 파악
- 시스템은 저장된 템플릿 규칙을 적용해 주간 기본 템플릿(평일 저녁 1회, 주말 점심·저녁 2회) 기준의 슬롯 목록을 동적으로 보여준다.
- 슬롯 목록은 날짜와 식사 시간대를 기준으로 정렬되어 사용자들이 가능한 시간을 한눈에 파악할 수 있어야 한다.
- 템플릿에 따라 생성된 슬롯만 표시되며, 수동으로 추가된 슬롯이 없음을 명확히 한다.

## 5. 개인 응답 조회와 수정 준비
- 닉네임을 등록하지 않은 사용자는 약속 정보만 확인할 수 있으며, 응답을 제출하려면 닉네임(및 설정한 경우 PIN)을 입력해야 한다.
- 이미 응답한 사용자가 동일한 닉네임(+PIN)을 입력하면 이전에 제출한 가용성 선택이 그대로 로드되어 수정 여부를 결정할 수 있어야 한다.
- 닉네임이 중복되거나 PIN이 일치하지 않는 경우 사용자가 다시 시도하거나 닉네임을 새로 등록하도록 안내한다.

## 6. 전체 가용 요약 이해
- 각 슬롯별 가능한 인원 수와 응답 제출 인원 대비 비율을 확인할 수 있어야 하며, 누구나 동일한 요약 데이터를 본다.
- 응답자 리스트나 세부 정보는 공개하지 않고, 집계 정보만 제공하여 개인정보 보호 요구사항을 따른다.
- 요약 데이터는 최신 응답을 반영해야 하며, 필요 시 사용자가 데이터를 새로 고칠 수 있음을 안내한다.

## 7. 오류 및 예외 상황 대응
- 약속이 존재하지 않을 때는 일반적인 "약속을 찾을 수 없음" 알림을 제공하고, 새로고침이나 링크 확인 외에 추가 액션을 요구하지 않는다.
- 서버 장애나 네트워크 오류 시에는 일시적인 문제임을 알리고, 나중에 다시 시도하라는 메시지를 보여준다.
- 템플릿 로딩 실패 등 예외 상황에서도 사용자 데이터가 손실되지 않도록 안내 메시지를 제공하고, 재시도를 통해 정상 슬롯 목록이 표시될 수 있음을 명시한다.
