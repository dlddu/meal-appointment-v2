# 구현 작업 지침

구현 작업을 수행하면서 작성하거나 수정하는 코드 및 문서는 이 지침을 따릅니다. 특히 구현 기록은 `agent/indexes/implementation-index.json`에 남깁니다.

1. 작업 요청에 명세가 포함되어 있지 않다면 먼저 관련 명세를 생성하거나 확인합니다. 필요한 경우 `agent/specs` 디렉터리 및 관련 인덱스를 참조합니다.
2. 구현은 명세에 정의된 범위 내에서 가장 단순하고 명확한 해결책으로 작성합니다. 추가 기능이 필요하면 명세를 갱신한 후 진행합니다.
3. 코드나 문서에는 어떤 명세를 근거로 구현했는지 주석 또는 머리글 형태로 명시합니다(예: `Implemented for spec: agent/specs/example.md`).
4. 구현이 완료되면 `agent/indexes/implementation-index.json`에 다음 정보를 추가합니다.
   - 관련 명세 파일 경로
   - 구현 결과물 경로 목록
   - 구현 요약 및 특이 사항
5. 명세 또는 구현이 변경되면 해당 인덱스 항목도 함께 갱신하여 최신 상태를 유지합니다.
