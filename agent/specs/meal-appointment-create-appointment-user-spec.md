# 식사 약속 생성 기능 사용자 명세

## 1. 목적과 범위
- 본 명세는 최종 사용자가 식사 약속을 생성할 때 경험하는 기능적 요구사항을 정의한다.
- 약속 생성 이후의 공유, 참여, 응답 수집 단계는 참조만 하며, UI/UX 레이아웃과 시각적 요소는 범위에서 제외한다.
- 본 명세는 `meal-appointment-domain-spec.md`의 약속 생성 흐름과 `meal-appointment-architecture-spec.md`의 API 계약을 기반으로 한다.

## 2. 사용자 전제 조건
- 사용자는 로그인 없이 웹 애플리케이션에 접근한다.
- 약속 생성자는 고유한 닉네임이나 계정을 요구받지 않는다.
- 애플리케이션은 단일 시간 슬롯 템플릿(주간 기본 템플릿)만 선택 가능하다고 안내한다. 추후 템플릿이 추가될 경우, 사용자는 제공되는 목록에서 1개를 선택한다.

## 3. 기본 시나리오
1. 사용자가 식사 약속 생성 페이지에 접속한다.
2. 사용자가 약속 정보를 입력하고 필수 항목을 채운다.
3. 사용자가 약속 생성을 확정하면 시스템이 약속을 저장하고 공유 가능한 링크를 반환한다.
4. 사용자는 제공된 링크를 복사하여 원하는 채널로 공유할 수 있다.

## 4. 입력 항목 및 유효성 규칙
| 항목 | 설명 | 필수 여부 | 유효성 기준 |
| --- | --- | --- | --- |
| 제목 | 약속의 이름 또는 주제 | 필수 | 1~60자, 앞뒤 공백 제거 후 빈 문자열 불가 |
| 설명 | 약속에 대한 간단한 안내 문구 | 선택 | 0~200자, 허용 길이 초과 시 오류 안내 |
| 시간 슬롯 템플릿 | 약속에 적용할 템플릿 식별자 | 필수 | 제공된 템플릿 중 정확히 1개 선택. 초기에는 "주간 기본 템플릿"만 노출 |

- 시스템은 입력 즉시 클라이언트 측에서 기본 검증을 수행하고, 제출 시 서버 검증을 통해 동일 규칙을 다시 확인한다.
- 서버 검증 실패 시, 사용자는 실패 사유(예: "제목은 60자를 넘을 수 없습니다")를 확인하고 수정 후 재시도할 수 있다.

## 5. 시스템 행동
- 사용자가 성공적으로 제출하면 시스템은 다음 데이터를 생성한다.
  - 고유 `appointment_id`
  - 사용자가 입력한 제목, 설명, 선택된 템플릿 식별자
  - 생성 타임스탬프
- 시스템은 생성된 `appointment_id`를 포함한 공유 URL(예: `/appointments/{appointment_id}`)을 즉시 사용자에게 제공한다.
- 서버는 추가 보조 토큰이나 비밀번호를 생성하지 않는다. 공유 URL만으로 참여자가 접근 가능하다.
- 동일한 제목과 설명의 약속을 여러 번 생성할 수 있으며, 각 생성 요청은 별도의 `appointment_id`를 가진다.

## 6. 성공 피드백
- 약속 저장이 완료되면 사용자는 성공 메시지와 공유 URL을 확인할 수 있어야 한다.
- 사용자가 링크를 복사하거나 다시 확인할 수 있도록 공유 URL은 약속 생성 직후와 페이지 재방문 시 모두 동일하게 조회 가능해야 한다.
- 시스템은 약속 세부 정보 페이지 진입을 위한 버튼 또는 링크를 제공하되, 구체적 레이아웃은 UI/UX 범위에 속하므로 명세에서 정의하지 않는다.

## 7. 오류 및 예외 처리
- 서버가 약속 생성을 거부하는 경우(유효성 오류, 서버 오류)는 사용자에게 원인과 해결 방법을 전달한다.
  - 유효성 오류: 필드별 오류 메시지 표시.
  - 서버 오류: "일시적인 문제로 약속을 생성할 수 없습니다. 잠시 후 다시 시도하세요."와 같은 일반 메시지.
- 네트워크 장애 발생 시, 사용자는 재시도 옵션을 제공받아야 하며, 재시도 시 중복 약속이 생성될 수 있음을 안내한다.
- 서버 응답을 받기 전 사용자가 페이지를 이탈하면 약속은 생성되지 않는다.

## 8. 보안 및 개인정보 고려 사항
- 로그인이나 별도 인증 없이 링크만으로 접근하는 모델임을 사용자에게 명시한다.
- 생성된 약속 정보에는 이메일, 전화번호 등 개인정보 입력을 요구하지 않는다.
- 시스템은 약속 생성 시 선택적 PIN이나 비밀번호를 설정할 수 있는 옵션을 제공하지 않는다.

## 9. 사후 조치 및 연계 기능
- 사용자는 공유 URL을 통해 참여자들이 닉네임으로 참여할 수 있음을 안내받는다.
- 약속 생성 직후, 시스템은 참여 현황을 추적할 수 있는 약속 세부 조회 경로로 이동하거나 이동 수단을 제공한다(구체적 UX 제외).
- 약속 삭제나 수정 기능은 이 명세 범위에서 다루지 않는다.
