# 명세 작성 작업 지침

이 지침은 명세 파일(`agent/specs/*`)과 명세 인덱스(`agent/indexes/spec-index.json`, `agent/indexes/spec-reference-index.json`)를 작성하거나 수정할 때 적용됩니다.

1. 사용자의 요구사항을 정확히 파악하여 요구사항의 언어로 명세 파일을 작성합니다. 필요 시 섹션 구조, 표, 다이어그램 등 요구사항에 적합한 형식을 선택합니다.
2. 사용자로부터 받은 체크리스트가 있다면 각 항목의 승인/반려 여부를 반영하여 최종 명세에 통합합니다.
3. 명세를 작성하거나 갱신한 후에는 `agent/indexes/spec-index.json`에 다음 정보를 기록합니다.
   - 명세 파일 경로
   - 명세의 의도 요약
   - 핵심 키워드 목록
   - 태그 3가지
4. 생성된 명세가 기존 명세와 연관되어 있다면 `agent/indexes/spec-reference-index.json`에 관계(예: 후속/선행/보완)를 명확히 기술합니다.
5. 명세 및 인덱스 파일은 JSON 포맷을 사용할 때 들여쓰기 2칸을 기본으로 하며, 키 이름은 스네이크 케이스를 권장합니다.
6. 명세가 변경되면 반드시 관련 인덱스 항목도 함께 갱신하여 일관성을 유지합니다.
