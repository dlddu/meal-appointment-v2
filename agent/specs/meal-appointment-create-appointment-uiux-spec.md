# 식사 약속 생성 페이지 UI/UX 명세

## 1. 개요
- 본 명세는 `meal-appointment-create-appointment-user-spec.md`의 요구사항을 기반으로, 식사 약속 생성 단일 페이지의 화면 구성, 시각 요소, 상호작용 규칙을 정의한다.
- 초점은 로그인 없이 접근하는 사용자의 간단한 약속 생성 경험이며, 약속 공유 이후의 흐름은 범위에서 제외한다.
- 명세에 정의된 디자인 토큰은 초기 베타 릴리스에서 재사용 가능한 기초 시스템으로 간주한다.

## 2. 주요 사용자 여정 요약
1. 페이지 진입 시 약속 생성 흐름과 템플릿 제약에 대한 안내를 확인한다.
2. 제목, 설명, 시간 슬롯 템플릿을 입력/선택한다.
3. "약속 만들기" 버튼을 눌러 제출한다.
4. 성공 시 공유 링크와 후속 안내를 확인하고 복사하거나 세부 정보 페이지로 이동한다.
5. 오류 발생 시 해당 메시지를 확인하고 수정 후 재시도한다.

## 3. 페이지 레이아웃
- **전체 레이아웃**: 폭 960px의 가운데 정렬 카드형 래퍼. 모바일(≤ 768px)에서는 좌우 16px 여백을 두고 전체 폭 사용.
- **구성 섹션**
  1. 상단 히어로 영역: 페이지 제목, 부제, 시각적 아이콘을 포함.
  2. 입력 폼 카드: 필수/선택 필드와 제출 버튼을 포함.
  3. 결과 패널: 제출 이후 공유 URL과 후속 행동 버튼 표시. 초기 상태에서는 접힌 상태.
  4. 정보 푸터: 비로그인 모델, 데이터 최소 수집 정책 등을 안내하는 간단한 텍스트 블록.
- **그리드 규칙**: 폼 내부는 단일 컬럼 스택. 데스크톱에서는 설명 필드가 4줄 높이의 텍스트 영역으로 확장되며, 템플릿 선택은 카드형 라디오 버튼 1열로 배치.

## 5. 핵심 컴포넌트 세부 규격
### 5.1 상단 히어로 영역
- **타이틀**: H1, 28px, 볼드, Primary Color.
- **부제**: 18px, Regular, Body Font, 최대 폭 640px.
- **아이콘 영역**: 64px 원형 배경(`Secondary Color` 20% 투명), 중앙에 식사 관련 라인 아이콘.
- **보조 텍스트**: 템플릿 제약 안내. 14px, `Secondary Color`를 사용.

### 5.2 입력 폼 카드
- 카드 폭: 100% (최대 720px), Surface Color 배경, Border Radius 적용, 내부 패딩 32px.
- 필드 순서: (1) 제목 입력, (2) 설명 텍스트 영역, (3) 시간 슬롯 템플릿 선택, (4) 네트워크 오류 안내 영역(초기 숨김), (5) 제출 버튼.
- 라벨: 14px, Medium, Heading Font, `#475569` 색상.
- 입력 필드: 100% 폭, 높이 48px, 내부 패딩 12px, Border Color 테두리. 포커스 시 Primary Color 2px 테두리 및 얇은 글로우.
- 에러 메시지: 필드 하단 12px, Error Color, 아이콘과 함께 좌측 정렬.
- 템플릿 선택: 라디오 버튼 대신 카드형 선택. 각 카드 높이 80px, 내부에 템플릿 이름, 설명, 배지. 선택 시 Primary Color 테두리와 배경 `rgba(42, 111, 151, 0.08)` 적용.
- 제출 버튼: Primary Color 배경, 흰색 텍스트, 52px 높이, 전체 폭. 로딩 시 내부 로더(원형 스피너)와 "생성 중..." 텍스트로 변경.

### 5.3 결과 패널
- 초기 상태: 폼 하단 접힘(토글 패널). 제출 성공 시 슬라이드 다운 애니메이션(200ms ease-out)으로 노출.
- 구성 요소: 성공 아이콘, 제목(20px Bold), 공유 URL 카드, "링크 복사" 보조 버튼, "약속 세부 정보로 이동" 기본 버튼.
- 공유 URL 카드: Surface Color, Border Color, 12px 패딩, 우측 끝에 복사 버튼(Secondary Color 테두리) 포함.
- 복사 성공 토스트: 화면 우상단, Success Color 배경, 3초 자동 닫힘.

### 5.4 오류 상태
- 서버 오류: 폼 카드 상단에 전체 폭 경고 배너 표시. Error Color 배경 10% 투명, 텍스트는 Error Color 18px Bold.
- 네트워크 오류 재시도 버튼: Secondary Color 배경, 흰색 텍스트, 버튼 높이 44px.

## 6. 상호작용 및 상태
- 입력 유효성 검사는 실시간(blur 시)과 제출 시 모두 수행. 오류 발생 시 해당 필드 포커스.
- 제출 중에는 버튼 비활성화 및 폼 필드 `disabled`. 로더 표시.
- 성공 패널 노출 후에도 사용자는 입력 값을 수정 가능하며, 수정 시 패널은 접히고 공유 URL은 숨김.
- 브라우저 뒤로 가기 시 경고 모달은 필요 없음(서버에서 생성 전까지 데이터 미저장).

## 7. 반응형 규칙
- 모바일에서는 히어로 아이콘 크기를 48px로 축소, 타이틀 24px.
- 폼 카드 패딩 20px로 축소, 버튼은 여전히 전체 폭.
- 템플릿 카드가 세로 스택으로 유지되며, 내용은 좌측 정렬.
- 공유 URL 카드에서 URL 텍스트는 2줄까지 허용, 이후 말줄임.

## 8. 접근성 고려사항
- 모든 상호작용 요소는 키보드 포커스 링(Primary Color 2px) 제공.
- 라벨과 입력은 `id/for` 연결 유지. 오류 메시지에는 `aria-live="polite"` 적용.
- 성공 및 오류 토스트는 `aria-live="assertive"`로 5초 내 자동 닫힘.
- 색 대비: Primary Color 텍스트는 배경 대비 비율 4.5:1 이상을 유지. Secondary Color는 텍스트에 직접 사용하지 않고 배경/배지용으로만 사용.

## 9. 콘텐츠 가이드라인
- 제목 플레이스홀더: "예: 팀 점심 모임".
- 설명 플레이스홀더: "약속 목적이나 참고 사항을 200자 이내로 입력하세요.".
- 템플릿 카드 설명: 기본 템플릿의 가용 요일 범위를 1줄(예: "월~금, 11:30-13:30")로 표기.
- 성공 메시지: "약속이 생성되었어요!" / 보조 텍스트: "아래 링크를 복사해 참여자를 초대하세요.".
- 오류 배너: "일시적인 문제로 약속을 생성할 수 없습니다. 잠시 후 다시 시도하세요.".

## 10. 샘플 디자인
- `/web-client/mockups/create-appointment.html`은 본 명세를 시각화한 정적 샘플이다.
- 샘플은 데스크톱 기준 화면(1440px 폭)에서 캡처한 스크린샷으로 제공되며, 실제 구현 시 React 컴포넌트와 CSS 모듈/스타일드 컴포넌트로 재현한다.
- 주요 요소:
  - 페이지 배경: 연한 블루-그레이 그라데이션.
  - 히어로 카드와 폼 카드가 세로로 정렬되며, 성공 패널이 하단에 접힌 형태로 포함.
  - Primary/Secondary 컬러 사용법, 버튼 스타일, 입력 필드 라벨링이 시각적으로 확인 가능.

## 11. 관련 명세 연계
- 본 UI/UX 명세는 사용자 입력 규칙을 정의한 `meal-appointment-create-appointment-user-spec.md`를 구체화하며, 동일한 필수/선택 필드 구조를 따른다.
- 약속 생성 이후 공유 링크 제공 흐름은 아키텍처 명세(`meal-appointment-architecture-spec.md`)의 API 응답 구조와 일치해야 한다.
