# 식사 약속 조회 기능 사용자 명세

## 1. 목적과 범위
- 본 명세는 최종 사용자가 공유 링크를 통해 식사 약속 세부 정보를 조회할 때 경험하는 요구사항을 정의한다.
- 약속 응답 생성·수정, 응답자 인증, 관리자 편집 기능은 본 명세 범위에 포함되지 않는다.
- 명세는 `meal-appointment-domain-spec.md`의 약속 객체 모델과 시간 슬롯 템플릿 규칙을 전제로 하며, UI 배치나 시각 디자인 세부 사항은 다루지 않는다.

## 2. 사용자 전제 조건
- 사용자는 약속 생성자가 공유한 URL(`/appointments/{appointmentId}`)을 통해 페이지에 접근한다.
- 접근 시 로그인, 계정 생성, PIN 입력이 요구되지 않는다. 닉네임 미등록 사용자도 약속 정보를 열람할 수 있다.
- 브라우저와 네트워크 환경은 표준 현대 웹 기술(ES2020, Fetch API)을 지원한다고 가정한다.

## 3. 기본 조회 흐름
1. 사용자가 공유 링크를 열면 시스템은 `appointmentId`를 이용해 약속 데이터를 조회한다.
2. 약속이 존재하면 조회 페이지가 로드되며, 약속 개요와 템플릿 기반 시간 슬롯 목록, 응답 요약, 응답자 상세 정보가 표시된다.
3. 사용자는 목록을 스크롤하며 원하는 정보를 확인한다. 읽기 전용 화면이므로 추가 입력이나 응답 제출은 요구되지 않는다.
4. 데이터 새로고침이 필요하면 사용자는 페이지 새로고침(브라우저 기본 기능) 또는 제공된 재시도 컨트롤을 이용해 최신 정보를 불러온다.

## 4. 표시 정보 요구사항
### 4.1 약속 개요
- 제목과 요약 설명을 필수로 보여주어 약속 목적을 즉시 이해할 수 있어야 한다.
- 약속 생성 시각, 마지막 업데이트 시각 등 메타데이터는 참고용으로 표시하되 응답 흐름과 직접 연결되지는 않는다.
- 약속이 존재하지 않을 때는 "약속을 찾을 수 없습니다"와 같은 안내 메시지를 제공하며, 로그인이나 추가 입력을 요구하지 않는다.

### 4.2 시간 슬롯 영역
- 시스템은 템플릿 정의(예: 주간 기본 템플릿)를 적용해 해당 약속에 유효한 슬롯 목록을 동적으로 구성한다.
- 슬롯 목록은 날짜(오름차순)와 식사 시간대 순으로 정렬된다.
- 각 슬롯에는 템플릿 이름과 설명이 함께 표시되어 사용자가 시간대 구성 규칙을 이해할 수 있다.
- 템플릿에서 파생된 슬롯만 노출되며 수동으로 추가된 슬롯이 없음을 명확히 안내한다.
- 템플릿 로딩 실패 시 사용자 데이터는 변경되지 않으며, 재시도 시 슬롯 목록을 다시 확인할 수 있음을 알려야 한다.

### 4.3 응답 요약 정보
- 각 슬롯별로 총 응답 수, 가능한 인원 수, 응답률 등의 집계 정보를 동일한 형식으로 제공한다.
- 응답 요약은 서버 최신 데이터를 반영해야 하며, 클라이언트 캐시가 오래된 경우 데이터를 새로 고칠 수 있는 방법(페이지 새로고침 또는 별도 "새로고침" 컨트롤)을 안내한다.
- 시스템 또는 네트워크 오류로 집계 데이터를 불러오지 못할 경우, 일시적 문제 메시지와 함께 재시도를 권장한다.

### 4.4 응답자 상세 정보
- 누구나 약속 조회 화면에서 응답자 리스트와 각 응답자의 슬롯별 선택 상태를 확인할 수 있어야 한다.
- 응답자 식별은 약속 응답 시 입력한 닉네임을 기준으로 하며, 별도의 개인정보(이메일, 전화번호 등)는 저장하지 않는다.
- 닉네임이 없는 사용자는 응답을 제출할 수 없음을 명확히 안내하며, 조회 화면에서는 응답 전 닉네임과 선택적 PIN을 입력해야 응답 제출이 가능하다는 점만 설명한다.

## 5. 상호작용 및 안내 메시지
- 조회 화면은 읽기 전용이다. 응답 제출 또는 수정 UI는 제공하지 않으며, 필요한 경우 별도 버튼이나 링크로 응답 작성 페이지로 이동하도록 안내한다.
- 닉네임을 등록하지 않은 사용자가 조회만 하는 경우 추가 조치를 요구하지 않는다. 응답 제출을 희망할 때 닉네임과 선택적 PIN 등록이 필요함을 안내 메시지나 FAQ로 제공한다.
- 기존 응답자의 응답 수정 흐름(닉네임 재입력, PIN 검증, 이전 선택 불러오기 등)은 본 명세 범위에 포함되지 않는다고 명시한다.

## 6. 데이터 새로고침 및 동기화
- 페이지 최초 로딩 시 서버 데이터와 동기화한다.
- 서버가 슬롯 집계를 계산해 제공하므로, 클라이언트는 주기적 폴링 없이 사용자가 요청할 때만 재조회한다.
- 사용자가 재시도를 요청하면 전체 약속 데이터를 다시 로드해 최신 응답 결과를 반영한다. 실패 시 기존 표시 정보는 유지하되, 오류 상태와 재시도 옵션을 함께 제공한다.

## 7. 오류 및 예외 처리
- 약속을 찾을 수 없는 경우: "약속을 찾을 수 없습니다" 메시지와 함께 공유 링크를 확인하라는 안내만 표시한다. 홈으로 이동, 새 약속 생성 등 추가 행동은 선택적으로 제공하되 필수는 아니다.
- 서버/네트워크 오류: "일시적인 문제로 약속 정보를 불러오지 못했습니다. 잠시 후 다시 시도하세요." 등의 안내와 재시도 버튼을 제공한다.
- 템플릿 로딩 실패: 슬롯 영역에 오류 메시지를 표시하고 전체 페이지를 새로고침하거나 재시도하면 슬롯을 다시 확인할 수 있음을 명시한다.

## 8. 보안 및 개인정보 고려 사항
- 공유 링크만으로 접근 가능한 공개 모델임을 페이지 내에서 안내한다.
- 시스템은 응답자 닉네임만 표시하며, 별도의 식별자나 연락처 정보는 저장하거나 노출하지 않는다.
- 닉네임 중복 또는 PIN 불일치와 관련된 검증 흐름은 응답 작성/수정 기능 범주에 속하므로 본 조회 명세에서는 정의하지 않는다.

## 9. 범위 외 기능 명시
- 응답 수정 기능, 닉네임 중복 검증, PIN 오류 처리, 이전 선택 불러오기 등은 조회 기능 범위에서 제외된다.
- 응답 제출 UI, 편집 가능한 슬롯 관리, 템플릿 변경 기능은 별도 명세에서 다룬다.
