# 식사 약속 참여 기능 사용자 명세

## 1. 목적과 범위
- 본 명세는 공유 링크를 받은 사용자가 닉네임 기반으로 약속에 참여하여 가능한 시간대를 제출·수정하는 기능적 요구사항을 정의한다.
- UI/UX 배치, 스타일, 상호작용 디자인은 범위에서 제외하며, 약속 생성/조회 기능은 연계 지점만 서술한다.
- 명세는 `meal-appointment-domain-spec.md`의 시간 슬롯 템플릿 규칙과 `meal-appointment-architecture-spec.md`의 공개 API 흐름을 전제로 한다.

## 2. 사용자 전제 조건
- 사용자는 생성자가 전달한 공유 URL(`/appointments/{appointmentId}`)을 통해 접근하며, 로그인이나 계정 생성이 필요 없다.
- 응답 제출을 위해 약속별 고유 닉네임이 필요하며, 선택적으로 PIN을 설정해 이후 수정 시 인증에 사용한다.
- 브라우저가 표준 웹 API(Fetch, LocalStorage)와 HTTPS 통신을 지원한다고 가정한다.

## 3. 기본 참여 흐름
1. 사용자가 공유 링크로 약속 상세 페이지에 접속해 약속 개요와 시간 슬롯 목록을 확인한다.
2. 사용자가 닉네임(필수)과 선택적 PIN(선택)을 입력해 참여 세션을 생성한다.
3. 시스템이 템플릿 규칙에 따라 생성한 슬롯 목록을 보여주면, 사용자가 가능한 슬롯을 선택한다.
4. 제출 시 선택한 슬롯은 "가능", 선택하지 않은 슬롯은 "불가능"으로 저장된다.
5. 제출 후 사용자는 동일한 링크와 닉네임(+PIN이 있는 경우 PIN)으로 재접속하여 응답을 수정할 수 있다.

## 4. 입력 항목 및 유효성 규칙
| 항목 | 설명 | 필수 여부 | 유효성 기준 |
| --- | --- | --- | --- |
| 닉네임 | 약속 내에서 사용자 식별에 사용 | 필수 | 1~30자, 앞뒤 공백 제거, 약속 내 중복 불가 |
| PIN | 응답 수정 시 본인 확인용 비밀값 | 선택 | 4~12자, 숫자/문자 조합 허용, 입력 시 서버에서 해시 저장 |
| 가능 슬롯 목록 | 사용자가 가능한 슬롯 ID 배열 | 필수 | 템플릿으로 생성된 슬롯 ID만 허용, 중복 없이 제출 |

- 닉네임 중복 제출 시 사용자는 "이미 사용 중인 닉네임" 안내와 함께 다른 닉네임을 선택해야 한다.
- PIN을 설정하지 않은 사용자는 이후 수정 시 PIN 입력을 요구받지 않는다.
- 서버 검증 실패 시, 필드별 오류 메시지를 제공하고 기존 입력을 유지한다.

## 5. 시스템 행동
- 참여 세션 생성 시 시스템은 `(appointmentId, nickname)` 고유성을 확인한 뒤 참가자 레코드와 선택적 PIN 해시를 저장한다.
- 가용성 제출 시 기존 응답을 덮어쓰며, 미선택 슬롯은 자동으로 `is_available=false`로 기록한다.
- 슬롯 식별자는 템플릿 규칙에서 파생된 문자열(예: `2024-05-14_dinner`)로 저장되며, 실제 슬롯 엔터티를 생성하지 않는다.
- 제출 후 시스템은 성공 메시지와 함께 사용자 선택 결과 및 전체 응답 요약을 즉시 조회 가능하게 해야 한다.

## 6. 재접속 및 수정 규칙
- 사용자가 동일 닉네임으로 재접속하면 이전 선택을 불러와 기본값으로 표시한다.
- PIN을 설정한 경우, 수정 시 동일 PIN을 요구하며 불일치 시 오류 메시지를 제공한다.
- 성공적으로 수정하면 기존 응답은 완전히 대체되며, 응답 집계가 즉시 갱신된다.

## 7. 오류 및 예외 처리
- 약속이 존재하고 만료되지 않은 경우에만 참여가 가능하며, 존재하지 않을 때는 "약속을 찾을 수 없습니다" 안내 후 추가 입력을 차단한다.
- 서버/네트워크 오류 발생 시 현재 입력을 보존하고 재시도 옵션을 제공한다.
- PIN 불일치, 닉네임 중복, 유효하지 않은 슬롯 ID 등 검증 오류는 구체적인 원인을 안내한다.

## 8. 보안 및 개인정보 고려 사항
- 로그인 없는 공개 모델임을 명시하고, 닉네임 외 개인 식별 정보는 요구하거나 저장하지 않는다.
- PIN이 있을 경우 서버는 해시로 저장하며 평문 PIN을 반환하지 않는다.
- 공유 링크만으로 접근이 가능하므로, 링크를 신뢰할 수 있는 대상에게만 전달하도록 안내한다.

## 9. 범위 외 기능 명시
- 약속 확정, 추천 시간대, 자동 알림, 계정 기반 인증은 본 명세 범위에 포함되지 않는다.
- 약속 생성자의 응답 삭제, 참여자 강퇴, 슬롯 수동 추가/삭제 기능은 제공하지 않는다.
