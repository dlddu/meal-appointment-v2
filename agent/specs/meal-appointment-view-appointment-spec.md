# 식사 약속 조회 기능 명세

## 1. 목적과 범위
- 본 명세는 생성된 식사 약속을 공유 링크(`/appointments/{appointmentId}`)로 조회할 때 사용자에게 제공되어야 하는 정보, 화면 구성, 백엔드 계약 상의 최소 요구사항을 정의한다.
- 로그인 없이 누구나 접근 가능한 공개 조회 경험만을 다루며, 응답 생성·수정·삭제 흐름은 범위에서 제외한다.
- 데이터는 기존 도메인 명세(`meal-appointment-domain-spec.md`)에서 정의한 약속, 시간 슬롯 템플릿, 응답 엔터티를 그대로 활용한다.

## 2. 접근 경로 및 읽기 전용 정책
- **URL 접근:** 사용자는 공유 링크(`/appointments/{appointmentId}`)를 통해 약속 상세를 직접 조회할 수 있어야 한다. 추가 인증 또는 세션 검사를 수행하지 않는다.
- **읽기 전용 안내:** 화면 상단에 "이 페이지는 약속 정보 확인 전용" 문구와 함께 응답을 제출하거나 수정하려면 별도 입력이 필요함을 명시한다.
- **닉네임 미등록 사용자:** 약속 정보만 열람 가능한 상태로 노출하며, 가용성 응답을 제출하려면 이후 단계에서 닉네임과 선택적 PIN을 등록해야 함을 안내한다.

## 3. 약속 기본 정보 표시
- **제목 및 요약:** 약속 제목과 요약을 히어로 섹션에 표시하여 약속 목적을 즉시 전달한다. 요약이 비어 있을 경우 "요약 정보가 없습니다"를 표시한다.
- **메타데이터:** 약속 생성 시각(ISO-8601)을 "생성: YYYY.MM.DD HH:MM" 형식으로 표기한다. 메타 정보는 참고용으로만 노출하며 상호작용 요소와 연결하지 않는다.
- **비공개 항목 없음:** 생성자가 추가로 숨긴 필드는 존재하지 않음을 명시해 모든 공유 링크 방문자가 동일한 정보를 본다.

## 4. 템플릿 정보 및 슬롯 생성 규칙
- **템플릿 설명:** 약속에 연결된 시간 슬롯 템플릿 이름과 설명을 별도 카드에 노출해 참가자가 슬롯 구성 규칙(예: 요일, 식사 시간대)을 이해하도록 한다.
- **동적 슬롯 구성:** 프런트엔드는 템플릿 정의를 기반으로 주간 기본 슬롯 목록을 동적으로 생성한다. 백엔드는 계산된 슬롯 배열을 응답 본문으로 제공하거나 템플릿 규칙을 전달해 클라이언트에서 재구성할 수 있어야 한다.
- **정렬 규칙:** 모든 슬롯은 날짜 오름차순 → 식사 시간대(아침/점심/저녁 등) 순으로 정렬한다. 동일 날짜 내에서는 템플릿에 정의된 우선순위를 따른다.
- **템플릿 파생 슬롯만 표시:** 수동으로 추가된 임시 슬롯은 지원하지 않으며, 템플릿에서 파생된 슬롯만 조회 화면에 나타난다. 템플릿 로직이 실패하면 오류 상태(섹션 7)를 따른다.

## 5. 응답 현황 및 참여자 정보
- **슬롯별 지표:** 각 슬롯 카드에 총 응답자 수, 가능(Yes) 인원 수, 불가(No) 인원 수, 미정(Maybe) 인원 수를 모두 노출한다. 지표는 방문 시점의 최신 데이터를 기반으로 하며, 5분 이상 캐시된 데이터를 사용할 경우 "최근 업데이트: HH:MM" 타임스탬프를 병기한다.
- **요약 섹션:** 화면 상단 또는 우측에 전체 응답률 요약(총 응답자 수, 가능률 상위 슬롯, 응답 대기자 수 등)을 제공한다. 요약에는 데이터 새로고침 버튼과 "응답 반영까지 최대 10초 지연" 안내를 포함한다.
- **참여자 세부 정보:** 모든 슬롯 아래에 응답자 목록을 표 형태로 제공하며, 각 응답자의 닉네임과 해당 슬롯에 대한 선택(Yes/No/Maybe)을 공개한다. PIN은 절대 노출하지 않는다.
- **집계와 개인 정보 동시 제공:** 집계 데이터와 응답자별 세부 선택을 동시에 노출하여 투명한 참여 상황을 전달한다. 닉네임이 중복되면 최근 응답 시각을 함께 표기해 구분한다.
- **데이터 동기화:** 사용자가 새로고침 버튼을 누르면 최신 응답을 다시 요청해 모든 지표와 목록을 갱신한다. 새로고침 실패 시 섹션 7의 오류 처리 흐름을 따른다.

## 6. 사용자 안내 및 추가 행동 유도
- **응답 유도:** 약속 정보 하단에 "내 가용성 제출하기" 버튼을 배치하고, 클릭 시 닉네임 및 PIN 입력 단계(별도 페이지 또는 모달)로 이동하도록 안내 문구를 표시한다.
- **재응답 흐름 배제:** 기존 닉네임 검증이나 응답 수정 기능은 조회 명세 범위에 포함되지 않음을 "기존 응답을 수정하려면 추후 제공될 편집 기능을 기다려 주세요" 문구로 명확히 전달한다.

## 7. 오류 및 예외 상태
- **약속 없음 (404):** 존재하지 않는 `appointmentId`일 경우 404 페이지를 렌더링하고 "약속을 찾을 수 없습니다" 메시지와 함께 홈으로 돌아가기 버튼만 제공한다. 추가 입력을 요구하지 않는다.
- **서버/네트워크 오류:** 백엔드 또는 네트워크 장애가 발생하면 정보 영역 대신 "일시적인 오류가 발생했습니다. 잠시 후 다시 시도해 주세요" 메시지와 재시도 버튼을 노출한다.
- **템플릿 로딩 실패:** 템플릿 정보나 슬롯 계산이 실패할 경우 사용자 데이터(닉네임 등)는 저장하지 않고, 오류 메시지와 함께 재시도 버튼을 제공한다. 재시도 시 동일 API를 다시 호출한다.
- **데이터 지연 안내:** 백엔드에서 최신 응답을 가져오지 못한 경우 "응답 갱신에 지연이 있습니다" 알림 배너를 표시하고, 문제가 지속되면 지원 채널로 문의하라는 안내를 포함한다.

## 8. 비기능 및 관측 요구사항
- **공개 캐싱 정책:** CDN/브라우저 캐시는 최대 30초로 제한하여 응답 지연을 줄이고, 서버는 ETag/Last-Modified 헤더를 제공해 조건부 요청을 지원한다.
- **로깅:** 조회 성공 시 `appointment.viewed` 이벤트를 기록하고, 포함 필드는 `appointmentId`, `viewerIpHash`, `slotCount`, `participantCount`, `refreshTriggered` 여부다.
- **성능:** 첫 화면 로딩은 2초 이내를 목표로 하며, 슬롯 수가 50개 이상일 경우 가상 스크롤 또는 구역별 접기를 고려한다.

## 9. 비범위(Out of Scope)
- 닉네임 검증, PIN 확인, 기존 응답 불러오기 등 응답 수정 흐름은 향후 명세에서 다룬다.
- 응답자 개인정보를 비공개 처리하거나 익명화하는 기능은 지원하지 않는다. 닉네임 기반 공개가 기본 정책이다.
- 약속 생성자만 접근 가능한 관리자용 상세 정보, 수동 슬롯 추가, 참가자 개별 알림 기능은 본 명세에 포함되지 않는다.
