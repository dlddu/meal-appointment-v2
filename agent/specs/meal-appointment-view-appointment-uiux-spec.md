# 식사 약속 조회 페이지 UI/UX 명세

## 1. 개요
- 본 명세는 `meal-appointment-view-appointment-spec.md`의 요구사항을 기반으로, 공유 링크로 접근하는 약속 조회 화면의 시각적 구조와 상호작용 패턴을 정의한다.
- 목적은 방문자가 응답 없이도 약속 전반을 이해하고, 자신의 가용성을 제출하도록 유도하는 정보를 명료하게 전달하는 것이다.
- 명세에 정의된 디자인 토큰은 향후 조회·응답 흐름에서 재사용할 수 있는 공통 시스템을 전제로 한다.

## 2. 사용자 여정 요약
1. 공유 링크로 페이지에 진입하면 약속의 핵심 정보와 읽기 전용 정책을 확인한다.
2. 템플릿 요약과 동적 슬롯 목록을 탐색하며 가용성 현황을 파악한다.
3. 참여자 응답 테이블을 통해 누가 어떤 선택을 했는지 확인한다.
4. "내 가용성 제출하기" 버튼을 눌러 별도 응답 입력 플로우로 이동한다.
5. 데이터 새로고침 또는 오류 재시도 흐름을 통해 최신 정보를 확인한다.

## 3. 페이지 레이아웃
- **뷰포트 폭 1440px 기준 12단 그리드**: 최대 콘텐츠 폭 1080px, 좌우 마진 각 180px. 그리드 단위 72px.
- **반응형 브레이크포인트**: 데스크톱 ≥ 1024px, 태블릿 768–1023px, 모바일 ≤ 767px.
- **상단 고정 알림 바**: 높이 48px, 읽기 전용 안내 문구와 정보 아이콘 포함. 데스크톱에서 전체 폭, 모바일에서는 상단에 접히는 토글 배너.
- **히어로 헤더 섹션** (그리드 12칸 전체 사용):
  - 왼쪽: 약속 제목(H1), 요약, 메타데이터.
  - 오른쪽: 응답 요약 카드(최대 4칸 폭)로 총 응답 수, 최고 인기 슬롯, 새로고침 버튼 배치.
- **본문 2컬럼 레이아웃** (데스크톱):
  - 메인 컬럼(8칸): 슬롯 리스트 아코디언, 각 슬롯 카드에 집계 지표 포함.
  - 서브 컬럼(4칸): 템플릿 정보 카드, 데이터 지연 배너, 도움말.
- **응답자 테이블 섹션**: 그리드 전체 폭 사용, 슬롯 리스트 아래 배치. 데스크톱에서 고정 헤더, 모바일에서는 좌우 스크롤 가능한 카드형 테이블.
- **하단 CTA 영역**: 전체 폭 카드. "내 가용성 제출하기" 기본 버튼(Primary), "기존 응답 수정 예정" 안내 텍스트 포함.
- **풋터**: 지원 채널, 개인정보 안내, 로깅 고지 텍스트.

## 5. 핵심 컴포넌트
### 5.1 읽기 전용 알림 바
- 배경: `rgba(42, 111, 151, 0.12)`.
- 텍스트: Primary Color, 14px Medium.
- 좌측에 20px 정보 아이콘, 우측에 "응답 제출 방법 보기" 링크.
- 모바일에서는 닫기 아이콘 제공, 닫힌 상태에서도 상단에 6px 두께 강조 바 유지.

### 5.2 히어로 헤더 카드
- 폭 100%, Surface Color, Radius Large, 내부 패딩 32px.
- 제목: 32px Bold, Accent Neutral → Primary Color 그라데이션 밑줄(4px).
- 요약: 18px Regular, 최대 폭 640px. 요약 없을 경우 플레이스홀더 문구를 Secondary Color 60% 투명도로 표시.
- 메타데이터: 14px, Accent Neutral 80% 불투명, 작은 시계 아이콘과 함께.
- 응답 요약 카드: Surface Alt 배경, Radius Medium, 내부 패딩 24px. 지표 칩 3개(총 응답자, 최고 인기 슬롯, 응답 대기자). 새로고침 버튼은 Secondary Color 아웃라인, 아이콘+텍스트.

### 5.3 슬롯 카드 리스트
- 각 슬롯 카드: Surface Color, Radius Medium, 세로 패딩 20px, 좌측에 날짜/요일 배지(Primary Color 10% 배경, 텍스트 Primary Color Bold).
- 내부 구조: 상단 행에 슬롯 이름과 시간 범위, 우측에 집계 배지 3개(Yes/Maybe/No). 집계 배지는 Pill 형태(높이 28px)로 Success/Warning/Error 컬러 사용.
- 중간 행: 진행 바 형태의 가능률 시각화(전체 폭, Success 컬러 기반). 가능률 계산 불가 시 스트라이프 패턴.
- 하단 아코디언: 응답자 요약 3명까지 미리보기, "자세히 보기" 버튼 클릭 시 전체 응답자 테이블 섹션으로 스크롤.
- 호버/포커스 시 Shadow Hover 적용, Border Strong 2px 테두리.

### 5.4 템플릿 정보 카드
- Surface Alt 배경, Radius Medium, 패딩 24px.
- 템플릿 아이콘(48px 원형 배경 Secondary Color 20% 투명)과 텍스트 블록.
- 템플릿 규칙 목록은 체크리스트 아이콘 + 텍스트 14px.
- 오류 상태(템플릿 로딩 실패) 시 카드 대신 Error Color 12% 배경의 경고 배너 표시.

### 5.5 응답자 테이블
- 데스크톱: 고정 헤더, zebra stripes(짝수 행 Surface Alt).
- 컬럼: 닉네임, 응답 상태, 최근 업데이트, 메모(향후 확장). 응답 상태는 색상 칩으로 표시.
- 모바일: 각 행을 카드로 변환, 헤더는 라벨-값 스택.
- 중복 닉네임 표시는 닉네임 옆에 타임스탬프 뱃지(Accent Neutral 60% 배경, 12px 텍스트)로 구분.

### 5.6 CTA 영역
- Surface Color, Radius Large, 패딩 28px, 중앙 정렬.
- 기본 버튼: Primary Color 배경, 56px 높이, 그림자 `0 8px 16px rgba(42, 111, 151, 0.25)`.
- 보조 텍스트: 14px, Accent Neutral. "기존 응답 수정 기능은 곧 제공됩니다." 문구 포함.

### 5.7 오류/빈 상태 패턴
- 404: 일러스트(128px)와 텍스트 센터 정렬, "홈으로 돌아가기" Secondary Color 아웃라인 버튼.
- 서버/네트워크 오류: Surface Color 배경, Error Color 16px Bold 제목, 14px 본문. 재시도 버튼은 Primary Color 아웃라인.
- 데이터 지연 배너: Warning Color 16% 배경, 왼쪽에 모래시계 아이콘, 텍스트 14px Medium, 닫기 버튼 제공.

## 6. 상호작용 및 상태
- 새로고침 버튼 클릭 시 로딩 스피너와 함께 버튼 텍스트 "새로고침 중"으로 변경, 10초 타임아웃 후 오류 토스트(에러 메시지) 표시.
- 슬롯 카드 클릭 시 해당 슬롯이 확장되어 응답자 테이블이 해당 슬롯 행으로 필터링됨(테이블 상단에 필터 배지 노출).
- 읽기 전용 알림 바 닫기 상태는 세션 스토리지에 저장하여 동일 세션 내 유지.
- CTA 버튼 클릭 시 페이드 아웃 애니메이션(150ms) 후 별도 페이지로 리디렉션.
- 토스트 패턴: 우상단 스택, 각 토스트 320px 폭, Radius Medium, 그림자 `0 10px 20px rgba(15, 23, 42, 0.16)`.

## 7. 반응형 및 접근성 지침
- 태블릿: 2컬럼 레이아웃 → 스택. 응답 요약 카드가 슬롯 리스트 위에 위치, 템플릿 카드가 하단으로 이동.
- 모바일: 모든 카드 전체 폭, 좌우 16px 패딩. 히어로 제목 24px, 요약 16px. 집계 배지는 24px 높이, 텍스트 12px.
- 테이블은 수평 스크롤 가능하도록 `overflow-x: auto` 적용, 각 행은 최소 폭 560px 유지.
- 포커스 링: Primary Color 3px 외곽선 + 2px 투명 아웃라인.
- 색 대비: 핵심 텍스트 대비비율 4.5:1 이상. Secondary Color는 텍스트로 직접 사용하지 않고 배경/아이콘용으로만 사용.
- 스크린 리더용 보조 텍스트 제공:
  - 읽기 전용 알림 바는 `role="status"`.
  - 집계 배지는 `aria-label="가능 5명"` 등 수치 포함.
  - 새로고침 버튼은 `aria-live="polite"` 영역 내 진행 상태 텍스트를 업데이트.

## 8. 콘텐츠 가이드라인
- 읽기 전용 문구: "이 페이지는 약속 정보 확인 전용이에요. 가용성을 제출하려면 아래 버튼을 이용하세요.".
- 메타데이터 포맷: "생성: 2024.05.12 11:30".
- 슬롯 라벨: "5월 20일 (월) · 점심 12:00-13:00" 형식.
- 집계 배지 텍스트: `가능 5`, `미정 2`, `불가 1`.
- 데이터 지연 배너: "응답 갱신에 지연이 있습니다. 잠시 후 다시 확인해 주세요.".
- 오류 메시지: "일시적인 오류가 발생했습니다. 잠시 후 다시 시도해 주세요.".
- CTA 버튼 텍스트: "내 가용성 제출하기".

## 9. 샘플 디자인
- `/web-client/mockups/view-appointment.html`은 본 명세를 시각화한 데스크톱 기준 정적 샘플이다.
- 샘플은 Primary/Secondary 컬러 사용, 2컬럼 레이아웃, 슬롯 카드/응답자 테이블 스타일을 포함한다.
- 스크린샷은 1440px 폭 뷰포트에서 캡처하며, 향후 구현 시 React 컴포넌트 구조로 분해 가능하다.

## 10. 관련 명세 연계
- 본 UI/UX 명세는 `meal-appointment-view-appointment-spec.md`의 정보 구조 요구사항을 시각적으로 구체화한다.
- 데이터 표기 방식과 새로고침 상호작용은 `meal-appointment-architecture-spec.md`에서 정의한 API 응답 및 관측성 요구사항을 반영한다.
- 색상 및 타이포그래피 토큰은 `meal-appointment-create-appointment-uiux-spec.md`와 일관성을 유지하여 제품 전반의 브랜드 경험을 통합한다.
