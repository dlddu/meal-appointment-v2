# 식사 약속 조회 화면 UI/UX 명세

본 문서는 `meal-appointment-view-appointment-user-spec.md`에 정의된 읽기 전용 조회 경험을 시각·인터랙션 관점에서 구체화한다. 사용자는 로그인 없이 공유 링크를 통해 접근하며, 약속 세부 정보와 응답 현황을 빠르게 확인하는 것이 목적이다.

## 1. 핵심 경험 원칙
- **즉시 파악**: 약속 제목, 설명, 주요 일정 정보를 뷰포트 상단에서 바로 확인할 수 있어야 한다.
- **명확한 정보 우선순위**: 슬롯 요약 → 응답자 상세 순으로 배치해 사용자에게 고수준 정보를 먼저 제공한다.
- **신뢰감 있는 상태 표현**: 데이터 새로고침, 오류, 비어 있는 상태를 명확한 메시지와 일관된 패턴으로 전달한다.
- **읽기 중심 인터랙션**: 클릭 가능한 요소는 새로고침/공유/응답 이동 등 필수 액션에 한정하며, 다른 영역은 스크롤 중심 탐색으로 유지한다.

## 2. 페이지 레이아웃
| 영역 | 설명 | 기본 폭 | 배경 | 주 컨텐츠 |
| --- | --- | --- | --- | --- |
| 상단 앱 바 | 로고/브랜드 명과 약속 공유 안내, 재시도 버튼 | 전체 가로 | 흰색 | 좌측: 브랜드, 우측: 재시도 & 공유 |
| 약속 개요 카드 | 약속 제목, 설명, 메타데이터(생성일, 마지막 업데이트) | 최대 960px | 흰색 카드 + 그림자 | 제목/설명/태그 배치 |
| 슬롯 요약 그리드 | 날짜-식사 시간대별 슬롯, 집계 정보 | 960px 이하에서 2열, 모바일 1열 | 연한 회색 배경 위 카드 | 슬롯 카드 각 요소 |
| 응답자 상세 섹션 | 응답자 리스트 및 슬롯별 응답 테이블 | 최대 960px | 흰색 | 탭 또는 아코디언 형태 |
| 푸터 안내 | 공개 링크 경고, 도움말 링크 | 전체 | 투명 | 작은 텍스트 |

### 2.1 레이아웃 동작
- 기본 컨테이너는 `max-width: 1120px`, 좌우 패딩 24px, 세로 구분 32px.
- 모바일(≤768px)에서는 모든 카드가 단일 컬럼으로 스택되고, 카드 내부 패딩은 16px로 축소한다.
- 테이블은 모바일에서 수평 스크롤이 아닌 카드형 리스트로 전환되도록 한다(반응형 변환 규칙은 5.3 참조).

## 3. 시각 디자인 토큰
| 토큰 | 값 | 용도 |
| --- | --- | --- |
| Primary 색상 | `#2E7D32` | 재시도 버튼, 강조 텍스트, 상태 아이콘 |
| Secondary 색상 | `#1565C0` | 공유 링크 버튼, 탭 활성 상태 |
| Neutral-100 | `#F5F7F8` | 페이지 배경 |
| Neutral-0 | `#FFFFFF` | 카드 배경 |
| Border 기본 | `#E0E3E7` (1px) | 카드 구분선, 표 헤더 |
| Success | `#1B5E20` | 슬롯 참여 가능 인원, 성공 메시지 |
| Warning | `#F9A825` | 낮은 응답률 강조 |
| Error | `#C62828` | 데이터 로딩 실패 메시지 |
| 본문 폰트 | "Pretendard", "Inter", sans-serif, 기본 16px | 일반 텍스트 |
| 제목 폰트 | "Pretendard", "Inter", sans-serif, H1 28px/H2 22px | 제목/섹션 헤더 |

- 타이포그래피 스케일: H1 28px/600, H2 22px/600, H3 18px/600, Body 16px/400, Caption 14px/400.
- 행간(Line-height)은 제목 1.3, 본문 1.6, 태그/캡션 1.4.

## 4. 주요 컴포넌트 사양

### 4.1 상단 앱 바
- 구성: 좌측 브랜드 라벨, 가운데 약속 제목(축약 표시), 우측 액션 버튼(`재시도`, `공유 링크 복사`, 선택적 `응답 작성하기`).
- 높이 64px, 배경 흰색, 하단 1px 그림자(`0 1px 2px rgba(15,23,42,0.08)`).
- 모바일에서는 제목이 2줄 허용, 액션 버튼도 바로 노출하되 버튼이 겹치지 않도록 행을 나눠 배치한다.

### 4.2 약속 개요 카드
- 상단에 텍스트 라벨 "약속 개요"(H3, Primary 색상).
- 제목(H1), 설명(Body), 메타 정보(아이콘 + Caption 텍스트) 순.
- 메타 정보 예: `⏱ 생성: 2024.05.01 09:20` / `🔄 마지막 업데이트: 2024.05.03 11:45`.
- 템플릿 안내 배지: `템플릿 기반 슬롯만 표시` 캡션 배지(Neutral-100 배경, Primary 텍스트).

### 4.3 슬롯 요약 카드
- 날짜 헤더: yyyy.MM.dd (요일) + 식사 시간대 배지.
- 집계 정보 행: `응답 N명`, `가능 M명`, `응답률 P%`. 값에 따라 강조 색상 사용.
  - 응답률 ≥ 70% → Primary 색상.
  - 40% ≤ 응답률 < 70% → Warning 색상.
  - 응답률 < 40% → Error 색상(단, 텍스트는 대비 확보 위해 진한 빨간, 배경은 `rgba(198,40,40,0.08)`).
- 카드 하단에 "템플릿: 주중 점심" 등 설명 텍스트.
- 모바일에서는 카드 폭 100%, 데이터 포인트는 두 줄로 재배치.

### 4.4 응답자 상세 섹션
- 탭 헤더: `응답자 목록`, `슬롯별 상세` 두 가지. 첫 탭 기본 활성.
- `응답자 목록` 탭: 리스트 아이템(닉네임, 마지막 응답 시각, 선택한 슬롯 배지 목록). 스크롤 가능.
- `슬롯별 상세` 탭: 테이블 형태 (`슬롯` 열 + 각 응답자 열). 데스크톱에서는 sticky 헤더, 응답자 이름은 회전(45°) 옵션 or 최소 폭 120px.
- 모바일 변환: 각 슬롯 카드 내에 응답자 선택 상태를 태그로 나열.

### 4.5 상태 메시지 패턴
- **로딩**: 컨텐츠 영역 스켈레톤(회색 애니메이션) + 상단 앱 바에 스피너.
- **오류**: 오류 아이콘(원형 경고 삼각형) + 메시지 + 재시도 버튼. 카드 상단 경고 배경(`rgba(198,40,40,0.08)`).
- **빈 상태**: 슬롯/응답자 데이터가 없으면 중립 아이콘 + "아직 응답이 없습니다" 텍스트.

## 5. 상호작용 및 접근성
- 버튼과 링크는 키보드 포커스 링(`outline: 3px solid rgba(21,101,192,0.45)`).
- 텍스트 대비 비율은 WCAG AA(4.5:1 이상) 준수.
- 재시도 버튼은 `aria-label="데이터 다시 불러오기"` 제공.
- 오류 알림은 `role="alert"` 적용, 스크린 리더에게 즉시 전달.
- 응답자 테이블은 `aria-describedby`로 슬롯 설명을 연결.

## 6. 반응형 기준
| 브레이크포인트 | 폭 | 주요 변경 사항 |
| --- | --- | --- |
| Desktop | ≥ 1024px | 슬롯 카드 3열, 응답자 테이블 가로 스크롤 지원 |
| Tablet | 768–1023px | 슬롯 카드 2열, 탭은 여전히 가로, 테이블은 최소 폭 720px |
| Mobile | ≤ 767px | 단일 컬럼, 탭은 슬라이더 형태, 테이블은 카드형 변환 |

## 7. 예시 흐름
1. 사용자가 공유 링크로 진입하면 스켈레톤이 표시되고 1초 내 데이터가 로드된다.
2. 약속 개요 카드가 상단에 등장하고, 사용자는 슬롯 요약 카드 그리드를 스크롤하면서 가용 슬롯을 파악한다.
3. 추가 정보가 필요하면 응답자 상세 탭으로 이동한다.
4. 최신 데이터가 필요할 경우 상단 `재시도` 버튼을 클릭해 전체 데이터를 재조회한다.
