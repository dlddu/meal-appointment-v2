# 식사 약속 참여 UI/UX 명세

본 문서는 `meal-appointment-participation-user-spec.md`의 흐름을 시각·인터랙션 관점에서 구체화한다. 공유 링크로 접근한 사용자가 닉네임 기반으로 참여 세션을 만들고, 주간 템플릿 슬롯에 대해 가용성을 제출·수정하는 경험을 정의한다. 날짜는 동적으로 무한 생성되며, 주간 기본 템플릿을 기준으로 탐색·선택 UI를 설계한다.

## 1. 핵심 경험 원칙
- **빠른 착수**: 페이지 진입 후 1~2개의 주요 입력(닉네임, 선택적 PIN)만으로 즉시 슬롯 선택을 시작할 수 있어야 한다.
- **주간 맥락 유지**: 주간 템플릿 단위로 그룹화된 슬롯을 한 화면에서 파악하고, 무한 스크롤/슬라이드로 앞뒤 주를 탐색할 수 있어야 한다.
- **명확한 상태**: 신규 참여, 기존 응답 불러오기, 제출 성공/오류, PIN 불일치 등 모든 상태를 일관된 토스트·인라인 메시지로 노출한다.
- **입력 보존**: 네트워크 오류나 닉네임 충돌 시에도 사용자가 선택한 슬롯과 입력값은 유지된다.

## 2. 페이지 레이아웃
| 영역 | 설명 | 최대 폭 | 배경 | 주요 요소 |
| --- | --- | --- | --- | --- |
| 상단 앱 바 | 브랜드, 약속 제목 요약, 도움말 아이콘 | 전체 | 흰색 | 좌측 브랜드, 우측 도움말/새로고침 |
| 참여 안내 히어로 | 약속 메타(제목, 설명), 공개 링크 경고 | 960px | 흰색 카드 + 그림자 | 제목/설명/배지/경고 텍스트 |
| 참여자 정보 카드 | 닉네임 입력, 선택적 PIN, 기존 응답 불러오기 상태 표시 | 640px | 흰색 카드 | 입력 필드, 상태 배지, 주의/성공 메시지 |
| 주간 슬롯 탐색 영역 | 주간 템플릿 그리드, 날짜 무한 탐색 내비게이션 | 1120px | 연한 회색 배경(`Neutral-100`) | 주간 헤더, 요일 열, 슬롯 버튼/배지 |
| 제출/요약 패널 | 선택 요약, 제출 버튼, 이전 응답 대비 변경점 | 960px | 흰색 카드 | 체크리스트, 토글, 제출 CTA |
| 푸터 | 공유 링크 주의, 접근성 안내 | 전체 | 투명 | 캡션 텍스트 |

### 2.1 레이아웃 동작
- 컨테이너 기본 `max-width: 1120px`, 양옆 패딩 24px, 섹션 간 간격 32px.
- 모바일(≤768px)에서는 카드가 단일 컬럼으로 스택되고 패딩은 16px로 축소된다.
- 주간 슬롯 영역은 모바일에서 수평 스크롤 가능한 슬라이드(요일 컬럼 단위)로 변환하며, 상단에 현재 주 범위 텍스트를 고정한다.

## 3. 시각 디자인 토큰
| 토큰 | 값 | 용도 |
| --- | --- | --- |
| Primary | `#2E7D32` | 제출 버튼, 선택된 슬롯, 배지 강조 |
| Secondary | `#1565C0` | 주간 내비게이션 화살표, 도움말 아이콘 |
| Neutral-100 | `#F5F7F8` | 페이지/슬롯 영역 배경 |
| Neutral-50 | `#EEF1F4` | 슬롯 버튼 비활성 배경 |
| Neutral-0 | `#FFFFFF` | 카드 배경 |
| Border | `#E0E3E7` 1px | 카드 구분선, 슬롯 테두리 |
| Success | `#1B5E20` | 제출 성공 텍스트/아이콘 |
| Warning | `#F9A825` | 응답 덮어쓰기 경고 |
| Error | `#C62828` | 검증 오류 메시지 |
| 폰트 | "Pretendard", "Inter", sans-serif | 기본 16px, H1 28px/600, H2 22px/600, Body 16px/400, Caption 14px/400 |

- 라인 높이: 제목 1.3, 본문 1.6, 캡션 1.4.
- 포커스 링: `outline: 3px solid rgba(21,101,192,0.45)`.

## 4. 주요 컴포넌트 사양

### 4.1 상단 앱 바
- 높이 64px, 하단 그림자 `0 1px 2px rgba(15,23,42,0.08)`.
- 우측 액션: `도움말`(모달 열기), `새로고침`(데이터 재조회). 모바일에서는 오버플로 메뉴(⋮)로 묶는다.

### 4.2 참여 안내 히어로
- H1 제목, Body 설명, 공개 링크 경고 배지(`Error` 텍스트 + `Neutral-100` 배경).
- 메타 텍스트: 생성일, 응답자 수, 템플릿 유형(예: "주중 점심 템플릿").
- 약속 메타 정보(생성일, 응답자 수, 템플릿 유형)를 요약 텍스트로 제공한다.

### 4.3 참여자 정보 카드
- 필드: 닉네임(Text), PIN(Password, 선택).
- 상태 영역: 기존 응답을 불러오면 `Success` 배지 + 마지막 제출 시각, 불일치 시 `Error` 메시지.
- 검증 규칙을 인라인 힌트로 표시: 닉네임 1~30자, PIN 4~12자. 오류 시 빨간 테두리 + 캡션 메시지.
- "현재 기기 저장" 토글을 제공해 LocalStorage에 닉네임/PIN을 보관하도록 안내(동의 텍스트 포함).

### 4.4 주간 슬롯 탐색 영역
- 상단 내비게이션: `◀ 이전 주`, `현재 주 범위(예: 2024.05.13~05.19)`, `다음 주 ▶`. 좌우 버튼은 Secondary 색상, 비활성 시 Neutral-50.
- 날짜는 백엔드 템플릿 규칙으로 무한 생성된다고 가정하고, 버튼 클릭 시 가상 리스트를 스무스하게 업데이트한다.
- 슬롯 그리드: 열=요일(월~일), 행=템플릿 식사 슬롯(예: 아침/점심/저녁). 카드형 버튼(높이 64px)으로 구성.
- 선택 상태
  - 선택됨: 배경 Primary, 텍스트 흰색, 좌측 체크 아이콘.
  - 비선택: Neutral-0 배경, 테두리 Border.
  - 비활성(템플릿에 없는 슬롯): Neutral-50 배경, `cursor: not-allowed`.
- 집계 배지: 각 슬롯 우측 상단에 `가능 인원 / 총 응답자` 캡션, 응답률에 따라 Success/Warning/Error 색상 적용.
- 모바일에서는 요일 헤더가 스냅 스크롤이 되며, 슬롯 버튼은 가로 플렉스 카드로 변환된다.

### 4.5 제출/요약 패널
- 선택한 슬롯 수, 선택하지 않은 슬롯 수, 마지막 제출 시각을 리스트업.
- 기존 응답이 있는 경우 "이전 선택 덮어쓰기" Warning 배지와 변경된 슬롯 개수 표시.
- 주요 CTA `가용 시간 제출`(Primary, 전체 폭), 보조 CTA `초기화`(텍스트 버튼).
- 제출 후 Success 토스트 + 패널 내 체크아이콘/문구 업데이트.

### 4.6 도움말/오류 토스트
- 토스트 위치: 우측 상단, 4초 후 사라짐, 포커스 이동 시 `aria-live="polite"`.
- 오류 토스트는 Error 색상 배경, 제출된 요청 ID(로그 추적용) 캡션을 포함.

## 5. 상호작용 흐름
1. 페이지 진입 시 스켈레톤 로딩 후 약속 메타와 템플릿 슬롯을 로드한다.
2. 닉네임 및 PIN(선택) 입력 → 참여 버튼 클릭 → 서버 중복 및 PIN 검증 성공 시 주간 슬롯이 활성화된다.
   - 기존 응답이 있는 닉네임인데 입력 PIN이 저장된 값과 다르면 참여 버튼 클릭 후 PIN 필드 하단에 오류 메시지와 오류 테두리를 노출한다.
3. 사용자는 슬롯 버튼을 탭/클릭하여 가능 상태를 토글한다. 선택은 즉시 요약 패널에 반영된다.
4. `가용 시간 제출` 클릭 시 서버에 배열을 전송하고, 성공하면 Success 토스트 + 제출 시각이 갱신된다.
5. 네트워크 오류 시 입력과 선택 상태를 그대로 유지하고, 토스트 + 제출 버튼 옆 재시도 아이콘을 표시한다.

## 6. 접근성 및 키보드 내비게이션
- 모든 입력·버튼에 명시적 `aria-label` 제공. 주간 내비게이션은 `aria-live="polite"`로 현재 주 범위를 알린다.
- 슬롯 버튼은 키보드 포커스 순서로 요일/시간 순회하며, Space/Enter로 토글 가능하다.
- 대비율 WCAG AA 준수, 포커스 링 일관 적용.

## 7. 반응형 기준
| 브레이크포인트 | 폭 | 변경 사항 |
| --- | --- | --- |
| Desktop | ≥ 1024px | 요일 7열 그리드, 제출/요약 패널 가로 배치 가능 |
| Tablet | 768–1023px | 요일 4열, 나머지는 슬라이드 스크롤; CTA는 전체 폭 |
| Mobile | ≤ 767px | 단일 열 카드, 주간 내비게이션을 상단 고정 바 형태로 축소 |

## 8. 샘플 디자인
- `agent/specs/samples/meal-appointment-participation-uiux-sample.html`은 위 명세를 반영한 데스크톱 기준 정적 목업이다.
- 샘플은 주간 템플릿(아침/점심/저녁)과 무한 주간 탐색 내비게이션을 포함하며, 스크린샷으로 레이아웃·색상·폰트를 확인할 수 있다.
